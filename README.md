# AI Recruiter — система проверки работ кандидатов

Веб-приложение для приёма и автоматической обработки работ кандидатов: парсинг файлов (PDF, DOCX, TXT и др.), извлечение заданий, оценка по критериям и эталонам, детекция схожести с другими работами и вероятности использования LLM, формирование отчётов для организаторов и кандидатов.

---

## Описание проекта

Система предназначена для организаторов конкурсов или приёмных кампаний. Кандидат загружает один файл с ответами на несколько заданий (1–4). Сервис:

- **Парсит** документ и разбивает текст на задания (в т.ч. с учётом изображений и OCR).
- **Очищает** ответы от «хвостов» (переносов текста между заданиями).
- **Строит эмбеддинги** (SBERT) и считает **схожесть с эталоном** (HR/AI) и **схожесть с уже загруженными работами**.
- **Оценивает задания 1–3** по системе eval_v6 (критерии + косинусное сходство с эталонами), **задание 4** — по логичности и оригинальности.
- **Оценивает вероятность использования LLM** (эвристика или BERT-score).
- **Формирует два отчёта:** подробный для организаторов (критерии, баллы, рекомендация) и краткий человечный для кандидата.

Данные хранятся в SQLite; интерфейс — веб (загрузка, таблица работ, отчёт по документу, экспорт в PDF).

---

## Этапы обработки файла

После загрузки файла выполняется последовательность этапов (прогресс отображается в реальном времени через WebSocket):

| № | Этап | Описание |
|---|------|----------|
| 1 | **Парсинг** | Извлечение текста (и при необходимости изображений) из файла (PDF, DOCX, TXT, MD, SQL, XLSX и др.). |
| 2 | **Извлечение заданий** | Разделение текста на блоки по заданиям 1–4 (по маркерам, заголовкам, структуре). |
| 3 | **Очистка** | Удаление «хвостов» — фрагментов, ошибочно попавших из одного задания в другое (по эмбеддингам и порогам схожести). |
| 4 | **Эмбеддинги** | Генерация векторных представлений для каждого задания и всего текста (SBERT локально или через API). |
| 5 | **Схожесть** | Расчёт схожести с эталонным файлом и с уже загруженными работами (для детекции копирования). |
| 6 | **Детекция читинга** | Оценка вероятности использования LLM (эвристика или BERT-score) и учёт схожести с другими работами. |
| 7 | **Оценивание** | Оценка заданий 1–3 по eval_v6 (критерии + эталоны HR/AI), задание 4 — по логике и оригинальности; расчёт среднего балла по 1–3; генерация отчётов для организаторов и кандидата. |
| 8 | **Генерация отчёта** | Подготовка данных отчёта (сохранение в БД). |
| 9 | **Завершение** | Статус «completed», документ доступен в таблице и для просмотра отчёта. |

При ошибке на любом этапе статус переводится в `error`; документ можно переобработать (reprocess).

## Основные страницы и API

| Страница / Endpoint | Описание |
|--------------------|----------|
| `/` | Главная — редирект на страницу загрузки. |
| `/upload` | [Загрузка файлов](http://localhost:5000/upload) — форма загрузки документа с ответами кандидата. |
| `/report` | [Таблица отчётов](http://localhost:5000/report) — список всех обработанных документов, фильтры, статистика (графики), действия: открыть отчёт, одобрить, заблокировать, удалить, экспорт в PDF, переобработка. |
| `/api/report/<doc_id>` | Открытие полного отчёта по документу (HTML): ответы по заданиям, критерии, баллы, схожесть с эталоном, % LLM, подробный отчёт и отчёт для кандидата, редактирование критериев и оценок. |
| `/api-docs` | [Документация API](http://localhost:5000/api-docs) — описание основных эндпоинтов для разработчиков. |
| `/health` | Проверка работоспособности сервиса (для мониторинга и деплоя). |

Краткое описание API: `GET /api/info` возвращает список маршрутов и поддерживаемых форматов.

## Запуск и настройка

### Требования

- Python 3.10+
- Установленные системные зависимости при необходимости: Tesseract (для OCR в PDF/DOCX), poppler (для pdf2image) — опционально.

### 1. Клонирование и виртуальное окружение

```bash
git clone <url-репозитория> ai_recruter
cd ai_recruter
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

Для локальной генерации эмбеддингов и локальных моделей (Qwen, BERT-score) при первом запуске могут подтягиваться большие модели — убедитесь, что есть место и при необходимости задайте `LOCAL_MODELS_DIR` и `HF_API_TOKEN` при использовании API.

### 3. Конфигурация (переменные окружения)

Создайте файл `.env` в корне проекта или задайте переменные в системе. Основные:

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `SECRET_KEY` | Секрет Flask (для сессий) | `dev-secret-key-change-in-production` |
| `DATABASE_PATH` | Путь к файлу SQLite | `data.db` |
| `EMBEDDING_METHOD` | Метод эмбеддингов: `sbert` или `qwen` | `sbert` |
| `API_PRIORITY` | Приоритет: `api_first` или `local_first` (для эмбеддингов/моделей) | `api_first` |
| `HF_API_TOKEN` | Токен Hugging Face API (для API-режима) | — |
| `ETALON_HR_JSON` | Путь к JSON эталона HR (задания 1–4) | `services/etalon_responses/etalon_hr.json` |
| `ETALON_AI_JSON` | Путь к JSON эталона AI | `services/etalon_responses/etalon_ai.json` |
| `LLM_LIKELIHOOD_METHOD` | Метод оценки LLM: `heuristic` или `bertscore` | `heuristic` |
| `BERTSCORE_MODEL` | Модель для BERT-score (например `cointegrated/rubert-tiny2`) | `cointegrated/rubert-tiny2` |

Полный список — в `services/config.py`. Файлы эталонов должны быть в формате JSON: массив объектов `[{"task_number": 1, "content": "..."}, ...]`.

### 4. Запуск приложения

**Локально (режим отладки):**

```bash
python app.py
```

Приложение будет доступно по адресу: http://localhost:5000 (главная перенаправит на `/upload`).

**Production (Gunicorn + Eventlet, например для Render):**

```bash
gunicorn -k eventlet -w 1 --timeout 180 -b 0.0.0.0:5000 wsgi:app
```

Порт можно задать переменной окружения `PORT` (на хостинге часто используется `-b 0.0.0.0:$PORT`).

## Структура проекта (кратко)

- `app.py` — точка входа Flask, маршруты, инициализация БД и сервисов.
- `wsgi.py` — точка входа для Gunicorn (eventlet monkey-patch + импорт приложения).
- `services/` — парсинг, очистка заданий, эмбеддинги, анализ схожести, eval_v6, оценивание, генерация отчётов, WebSocket.
- `parsers/` — парсеры по типам файлов (PDF, DOCX, TXT, MD, SQL, XLSX и др.).
- `extractors/` — извлечение заданий из текста.
- `utils/` — БД (SQLite), логирование, эмбеддинги, детекция читинга, работа с файлами.
- `templates/` — HTML-шаблоны (загрузка, отчёт, таблица работ, API docs).
- `static/` — CSS, JS, шрифты, загруженные изображения и экспортированные PDF.

База данных создаётся автоматически при первом запуске; миграции для новых колонок выполняются при инициализации `Database`.
